#  Go Concurrency Learning Roadmap

Мой путь изучения конкурентности в Go: от простых каналов до сложных конвейеров (pipelines). В этом репозитории собраны практические проекты, которые я реализовал.

##  Этапы обучения

### 1. Основы и Синхронизация (`The Task Force`)
**Цель:** Понять, как запускать горутины и ждать их завершения.
- Использование `sync.WaitGroup`.
- Работа с небуферизированными каналами.
- Понятие блокировки (blocking).

### 2. Паттерн "Диспетчер" (`Dispatcher`)
**Цель:** Реализовать распределение задач.
- Передача задач от одного отправителя к одному воркеру.
- Понимание цикла `for range` для чтения из канала до его закрытия.

### 3. Пул воркеров и Сбор данных (`Image Processor Simulator`)
**Цель:** Самый сложный этап — создание эффективного конвейера.
- **Worker Pool:** Запуск нескольких параллельных воркеров для ускорения работы.
- **Pipeline:** Передача данных через цепочку `Generator -> Tasks -> Workers -> Results`.
- **Close Signaling:** Использование анонимных горутин для своевременного закрытия каналов.
